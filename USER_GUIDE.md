# SantiPX 小白使用说明书（业务用户版）

这份说明给实际使用系统的人看，不讲服务器部署。

## 1. 先记住一句话

你只需要按这个顺序操作：

```text
打开链接 -> 配置 API -> 上传产品图 -> 填写需求 -> 生成节点 -> 批量生图 -> 导出素材
```

## 2. 打开系统

公司平台会给你一个入口链接，正常链接会带 `phone` 参数，例如：

```text
https://px.ks.santisaas.com/?phone=10000000001
```

这里的 `10000000001` 是示例号码，不是真实手机号。

系统会用手机号区分用户。不同手机号之间的数据不会混用。

如果打开页面后弹出“请输入手机号”，说明当前链接没有带手机号。你只需要输入自己的手机号，然后点击“进入”。

## 3. 页面上有什么

系统主要有两个入口：

- 品宣：上传产品图、填写需求、生成宣传图节点、批量生图、导出结果
- 设置：配置 API 供应商、服务映射、批量生图并发数

电脑端通常左侧有图标入口。

手机端通常底部有图标入口：

- 喇叭图标：品宣
- 滑杆图标：设置

## 4. 第一次使用前先配置 API

如果页面提示“部分服务未配置”，说明还没有配好 API。

点击“设置”，按下面步骤配置。

### 4.1 添加供应商

点击“添加供应商”。

常见填写方式：

- 平台：选择“自定义”
- 名称：随便写一个你能看懂的名字，例如“我的生图接口”
- Base URL：填写你的接口地址，例如 `http://0.0.0.0:8000/v1`
- API Key：填写你的 Key
- 模型：填写可用模型名，多个模型可以按界面要求添加

填写完成后保存。

### 4.2 配置服务映射

在“品宣服务映射”里，需要至少配置两个功能：

- 品宣规划：用于分析产品图和创作意图，生成宣传图节点
- 品宣生图：用于把宣传图节点生成真正的图片

每个功能都要选择对应的供应商和模型。

如果这两个都配置好了，“部分服务未配置”的提示就会消失。

### 4.3 并发生成数

“并发生成数”控制一次同时生成几张图。

新手建议先用：

```text
1
```

如果你的接口稳定、额度充足，再调大。

并发太高可能会导致接口限流或部分图片失败。

## 5. 开始做一组宣传图

点击“品宣”回到主工作区。

### 5.1 上传产品参考图

找到“产品参考图”，点击“上传图片”。

建议上传：

- 产品主图
- 包装图
- Logo 图
- 材质细节图
- 使用场景图

上传后可以给每张图选择用途，例如“主产品”“包装”“Logo”“材质细节”“使用场景”。

如果你只上传一张图，也可以正常使用，但 AI 对产品理解可能没有多图准确。

### 5.2 填写创作意图

找到“创作意图 / Prompt 文案”。

这里写你想做什么图，不需要写得很专业。

示例：

```text
我想要在淘宝上卖这款陶瓷办公茶杯，需要一组图片来帮我展示这个茶杯的用途以及这个茶杯很好用、有钩子，让用户都想来买。介绍图要包含各种场景下这个杯子的使用场景。
```

你可以写：

- 产品卖给谁
- 想突出什么卖点
- 想用于哪里，例如淘宝、抖音、小红书、详情页
- 想要什么风格，例如高端、干净、生活方式、强促销
- 不想出现什么内容

## 6. 生成宣传图节点

产品图和创作意图都准备好后，点击“生成节点”。

系统会自动做几件事：

1. 分析产品图
2. 理解产品卖点
3. 生成一组宣传图节点
4. 每个节点会包含标题、卖点、构图说明和生图提示词

生成完成后，中间区域会出现多张“宣传图节点卡片”。

## 7. 检查和调整节点

每个节点代表一张要生成的宣传图。

你可以检查：

- 标题是否符合你的目标
- 卖点是否准确
- 提示词是否跑偏
- 是否需要删掉某个节点
- 是否需要手动添加节点
- 是否需要重新排序

常用操作：

- 添加节点：手动补一张图的需求
- 重编排：让系统重新整理节点顺序和内容
- 删除：删除不需要的节点
- 上移/下移：调整节点顺序

如果你不确定怎么改，可以先直接生图，看效果后再回来调整。

## 8. 生成图片

### 8.1 单张生成

在某个节点卡片上点击生成图片相关按钮，可以只生成这一张。

适合先测试效果。

### 8.2 批量生图

点击“批量生图”，系统会按节点批量生成图片。

如果有些成功、有些失败，通常是这些原因：

- API 接口限流
- 并发数太高
- 某个提示词触发接口限制
- 自定义供应商返回了文字而不是图片
- 网络不稳定

处理建议：

1. 先把“并发生成数”调成 `1`
2. 失败的节点单独重试
3. 如果一直失败，检查“设置”里的供应商模型是否是图片生成模型
4. 检查 API Key 是否可用、余额是否足够

## 9. 查看生成结果

生成成功后，节点卡片右侧会显示图片。

你可以查看：

- 产品是否像原图
- 卖点是否表达出来
- 文字是否正确
- 构图是否适合电商
- 是否有多余元素
- 是否需要重试

如果图片不满意，可以：

- 修改节点提示词后重新生成
- 修改创作意图后重新生成节点
- 上传更多参考图后重新生成

## 10. 导出结果

图片生成完成后，点击“批量导出”。

导出内容通常包括：

- 生成的宣传图
- 节点信息
- 提示词内容

如果你只想保存图片，也可以在图片预览处单独保存。

## 11. 手机端怎么用

手机端和电脑端功能一样。

为了不拥挤，手机端有些卡片默认折叠。

常见折叠区域：

- 产品参考图
- 创作意图 / Prompt 文案
- 生成设置

需要编辑时，点开对应卡片即可。

底部图标：

- 喇叭：回到品宣工作区
- 滑杆：进入设置

## 12. 每个用户的数据是否隔离

隔离。

系统按 `phone` 区分用户。

例如：

```text
?phone=10000000001
?phone=10000000002
```

这两个用户的数据不同，包括 API 供应商也不同。

所以如果你换了手机号进入，看到的数据可能会变成空的，这是正常现象。

## 13. 常见提示说明

### 13.1 部分服务未配置

意思是 API 还没配完整。

解决方法：

1. 进入“设置”
2. 添加 API 供应商
3. 确认供应商有 API Key
4. 在“品宣服务映射”里配置“品宣规划”和“品宣生图”

### 13.2 还没有宣传图提示词节点

意思是你还没有生成节点。

解决方法：

1. 上传产品参考图
2. 填写创作意图
3. 点击“生成节点”

### 13.3 图片生成失败

先不要慌，常见原因是接口问题，不一定是你操作错了。

可以这样处理：

1. 降低并发生成数到 `1`
2. 单独重试失败节点
3. 检查 API Key
4. 检查模型是否支持图片生成
5. 如果返回“文本而不是图片”，说明供应商接口没有真正执行图片工具，需要换支持图片输出的模型或接口

### 13.4 页面让我输入手机号

说明当前链接没有带 `phone` 参数。

输入自己的手机号即可。

不要输入别人的手机号，否则会进入别人的数据空间。

## 14. 推荐新手操作流程

第一次使用可以照这个流程：

1. 打开公司平台给你的链接
2. 如果弹手机号，输入自己的手机号
3. 进入“设置”
4. 添加 API 供应商
5. 配好“品宣规划”和“品宣生图”
6. 回到“品宣”
7. 上传 1-5 张产品图
8. 填写创作意图
9. 点击“生成节点”
10. 检查节点内容
11. 先单张生成一张看看效果
12. 效果没问题后点击“批量生图”
13. 生成完成后点击“批量导出”

## 15. 写创作意图的小技巧

越具体，效果越好。

不推荐只写：

```text
帮我生成宣传图
```

推荐写：

```text
我要给这款产品做淘宝详情页宣传图，目标人群是办公室人群。请突出产品质感、日常使用场景、核心卖点和购买理由。画面要干净、高级、有电商海报感，不要像普通白底产品图。
```

可以包含这些信息：

- 目标平台：淘宝、京东、抖音、小红书
- 目标人群：宝妈、学生、办公室人群、礼品客户
- 核心卖点：保温、防尘、便携、大容量、高颜值
- 场景：办公室、厨房、客厅、户外、礼盒包装
- 风格：高级、清新、科技感、强促销、生活方式
- 禁止内容：不要换产品颜色、不要改 Logo、不要出现人物、不要虚假文字

## 16. 什么情况需要找管理员

遇到下面情况，可以找管理员：

- 页面打不开
- 一直提示网络错误
- API Key 不知道从哪里拿
- 不知道应该选哪个模型
- 所有节点都生成失败
- 导出失败
- 公司平台跳转进来没有带手机号
